# Обзор системы

## Назначение

**График заместителей** — внутреннее веб-приложение администрации Саратовской области для планирования и отображения мероприятий пяти заместителей губернатора. Пользователи видят календарную сетку по дням, создают и редактируют события, назначают участников из CRM, прикрепляют файлы и получают уведомления.

## Зачем отдельное приложение

Legacy CRM (`crm.admsr.ru`) — монолит на PHP. График вынесен в отдельный SPA на современном стеке, потому что:

- нужен отзывчивый интерфейс с горизонтальной прокруткой по дням и drag-scroll;
- события хранятся локально в SQLite (быстрые запросы, независимость от нагрузки CRM);
- участники и аутентификация по-прежнему берутся из CRM через HTTP или MySQL;
- в меню CRM остаётся только ссылка на grafic.

## Ключевые возможности

| Модуль | Описание |
|--------|----------|
| **Общий график** | Все заместители на одной странице (`/`) |
| **Индивидуальные графики** | Отдельная страница на каждого зама (`/marcenkovskiy`, `/markova`, …) |
| **Мероприятия** | CRUD: дата, время, место, тема, «весь день», скрытое, завершено |
| **Участники** | Сотрудники из CRM (`user`); поиск и мультивыбор |
| **Вложения** | Загрузка файлов до 25 МБ с проверкой magic bytes |
| **Скрытые мероприятия** | Видны в сетке (время + плашка), детали скрыты без прав |
| **Уведомления** | In-app + email при создании/изменении/удалении; напоминание за 5 минут |
| **Журнал действий** | Аудит для администраторов (`/logs`) |
| **Тема оформления** | Персональные настройки Nuxt UI (цвет, режим) |

## Технологический стек

### Фронтенд

- **Vue 3.5** + **TypeScript** + **Vite 7**
- **Vue Router 5** (file-based routes) + **vite-plugin-vue-layouts**
- **Nuxt UI v4** + **Tailwind CSS 4**
- **@vueuse/core**, **date-fns**, **zod**

### Бэкенд

- **Fastify 5** + **TypeScript**
- **SQLite** (`better-sqlite3`, WAL) — события, пользователи, логи, уведомления
- **MySQL** (`mysql2`, опционально) — прямой доступ к CRM
- **JWT** в HttpOnly cookie, **bcryptjs**, **zod**

### Сборка и деплой

- **pnpm** workspace (корень + `server/`)
- **nginx** — статика + reverse proxy
- **systemd** — `grafic-api.service`

## Роли пользователей (кратко)

| Локальная роль | Кто это | Права |
|----------------|---------|-------|
| `admin` | Администратор графика | Все графики, журнал, CRM-пользователи (API) |
| `manager` | Заместитель | Редактирование своего графика |
| `assistant` | Помощник зама | Редактирование графика «своего» зама |
| `moderator` | Модератор | Редактирование назначенных графиков |
| `user` | Исполнитель / руководитель CRM | Только просмотр |

Подробнее: [Права доступа](./07-permissions.md), [Аутентификация](./06-authentication.md).

## Маршруты SPA

| Путь | Страница | Доступ |
|------|----------|--------|
| `/` | Общий график | Авторизованные |
| `/{substitute}` | График зама | Авторизованные |
| `/login` | Вход | Публичный |
| `/logs` | Журнал действий | Только admin |
| `/customers` | CRM-пользователи | Заблокирован (редирект на `/`) |

Шаблонные страницы (`/inbox`, `/settings/*`) из Nuxt UI Dashboard не используются в навигации.
